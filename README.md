# FastAPI Test

A FastAPI backend project contains REST API with users, posts, authentication, and voting.

The project implements a simple social-post API: users can register, log in, create posts, update or delete their own posts, and vote for posts. PostgreSQL is used for persistence, SQLAlchemy handles database models and sessions, and JWT bearer tokens protect the routes that require authentication.

## Features

- User registration with password hashing
- Login with OAuth2 password flow
- JWT access token generation and validation
- CRUD operations for posts
- Owner-based authorization for protected post actions
- Search, limit, and offset parameters for post listing
- Voting system with duplicate-vote protection
- PostgreSQL database models with SQLAlchemy relationships
- Environment-based configuration through `.env`
- Interactive API docs generated by FastAPI

## Tech Stack

- Python 3.12
- FastAPI
- Uvicorn

- SQLAlchemy

- python-jose

## Installation

Clone the repository:

```bash
git clone https://github.com/aksenkkka/FastAPI-test.git
cd FastAPI-test
```

Create and activate a virtual environment:

```bash
python -m venv .venv
source .venv/bin/activate
```

Install dependencies:

```bash
pip install fastapi uvicorn sqlalchemy psycopg2-binary pydantic-settings python-jose passlib bcrypt email-validator
```

## Running the App

```bash
uvicorn app.main:app --reload
```

The API will be available at:

```text
http://127.0.0.1:8000
```

FastAPI documentation:

```text
http://127.0.0.1:8000/docs
```

Alternative documentation:

```text
http://127.0.0.1:8000/redoc
```

## API Endpoints

### Auth

```http
POST /login
```

Authenticates a user and returns a bearer token.

### Users

```http
POST /users/
GET /users/{id}
```

Creates a user and returns public user information without exposing the password.

### Posts

```http
GET /posts/
POST /posts/
GET /posts/{id}
PUT /posts/{id}
DELETE /posts/{id}
```

Post creation, update, and deletion require a valid JWT token. Update and delete operations also check ownership, so users cannot modify posts created by another account.

The post list supports:

```text
limit
skip
search
```

Example:

```http
GET /posts/?limit=10&skip=0&search=fastapi
```

### Votes

```http
POST /vote/
```

Adds or removes a vote for a post.

Example request:

```json
{
  "post_id": 1,
  "dir": 1
}
```

Use `dir: 1` to add a vote and `dir: 0` to remove it.
